# Синтаксис шаблонов

## Плейсхолдеры `[[...]]`

Подстановка значений из контекста, переданного через `template.data([name, value])`.

- Простая переменная: `[[userName]]`
- Вложенность по атрибутам/ключам: `[[obj.field]]` (если значение — объект с атрибутом или словарь с ключом).

Служебные корневые токены (часто **ЗАГЛАВНЫМИ**): `CELL`, `ITEM`, `TABLE`, `LIST`, `ENUM`, `MEDIA`, `PYDANTIC`, `SCHEMA`, `SQL`, `ORM` — см. [placeholders.md](placeholders.md).

## Циклы `%%len=VAR%%`

Тело между открывающей строкой `%%len=VAR` и закрывающей `%%` повторяется для каждого элемента последовательности `VAR` в контексте.

Внутри цикла доступны `[[ITEM]]`, `[[CELL.INDEX]]`, `[[CELL.ONE]]`, `[[CELL.COUNT]]`, а также `[[VAR.KEYS]]`, `[[VAR.DIVIDER]]`, `[[VAR.VALUES]]` для табличных строк.

## Слоты `@@name@@`

Места вставки **произвольного текста** после полного рендера плейсхолдеров и циклов.

- Значения задаются в Python: `template.slot("name", "текст")` или `render(extra={"__slot__name": "текст"})`.
- Вставленный текст **не** парсится повторно как шаблон (нет второго прохода по `[[...]]` внутри слота).

Порядок: сначала `%%len%%` и `[[...]]`, затем замена всех `@@...@@`.

Примеры **готового** Markdown после рендера: [rendering-samples.md](rendering-samples.md).
