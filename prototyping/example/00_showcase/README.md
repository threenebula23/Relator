# Пример 00: витрина функционала

Один скрипт демонстрирует основные возможности relator в одном Markdown-отчёте:

- скаляры `[[project]]`, `[[version]]`;
- слот `@@intro@@` с произвольным Markdown;
- `[[TABLE.*]]`, `[[TABLE.*.NUMBERED]]`;
- `[[LIST.*]]`, `[[ENUM.*]]`;
- цикл `%%len=rows%%` с `[[rows.KEYS]]` / `VALUES` / `DIVIDER`, `[[ITEM]]`, `[[CELL.*]]`;
- `[[MEDIA.*]]` (PIL `Image`).

## Запуск

Из корня репозитория `reporting` (с установленным пакетом или `PYTHONPATH=src`):

```bash
python prototyping/example/00_showcase/build_report.py
```

Отчёт: `dist/showcase_report.md` (каталог `dist/` создаётся рядом со скриптом).

Рядом появится каталог `assets/` с PNG для вставки медиа.

## Как выглядит вывод (после генерации)

Ниже тот же текст, что попадает в `showcase_report.md` (путь к картинке у вас будет локальный).

```markdown
# Витрина возможностей relator

Проект: **demo-app**, версия шаблона: `1.0`.

## Слот (произвольный Markdown после плейсхолдеров)

Это **Markdown** внутри слота: можно `код`, списки и даже [[псевдо-плейсхолдер]] без подстановки движком.

## Таблица (TABLE)

|sku|qty|
|---|---|
|A1|3|
|B2|1|

## Таблица с номером строки (TABLE.NUMBERED)

|#|sku|qty|
|---|---|---|
|1|A1|3|
|2|B2|1|

## Маркированный список (LIST)

- markdown
- ci
- tables

## Нумерованный список (ENUM)

1. Собрать JSON
2. Вызвать Template
3. Проверить dist/

## Цикл с таблицей и ITEM / CELL

|name|role|
|---|---|
|Alice|dev|
- Элемент: **name: Alice, role: dev** — позиция 1 из 2 (индекс 0).
|name|role|
|---|---|
|Bob|qa|
- Элемент: **name: Bob, role: qa** — позиция 2 из 2 (индекс 1).

## Медиа (MEDIA)

![logo](assets/img_<хеш>.png)
```

(Имя файла PNG стабильно при неизменных байтах изображения; путь абсолютный в локальном запуске — в отчёте будет ваш каталог `assets/`.)

Обратите внимание: внутри слота строка `[[псевдо-плейсхолдер]]` **не** стала плейсхолдером — слот подставляется после фазы `[[...]]`.

## Зачем этот пример

Быстро сравнить шаблон `template.md` с готовым видом отчёта и перенести приёмы в свой проект.

Статичные мини-примеры вывода по каждому типу плейсхолдера — в [../../docs/rendering-samples.md](../../docs/rendering-samples.md).
