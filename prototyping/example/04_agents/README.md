# Пример: встраивание вывода агента (CI / Cursor)

## Как это работает

1. Агент (или скрипт) сохраняет структурированный JSON: краткое резюме, риски, список задач.
2. `build_report.py` передаёт JSON в `Template` и заполняет **слоты** `@@summary@@`, `@@risks@@` — туда можно класть произвольный Markdown без экранирования плейсхолдеров шаблона.
3. Табличная часть (`[[TABLE.tasks]]`) остаётся строго данными из JSON.

Схема входного JSON: [agent_output.schema.json](agent_output.schema.json).

```bash
python build_report.py
```

Результат: `dist/agent_report.md`.

## Пример вывода

```markdown
# Отчёт агента

## Резюме агента

- Код соответствует стилю.
- Рекомендованы мелкие правки в обработке ошибок.

## Риски

- Риск регрессии в модуле `auth` при деплое без миграции.

## Задачи

|id|title|status|
|---|---|---|
|1|Добавить валидацию|open|
|2|Обновить тесты|done|
```

## Почему это удобно

- Слоты отделяют «сырой» текст агента от структуры отчёта.
- Один и тот же шаблон для человека и для CI-комментария.

## Экономия времени

Ручное перенесение вывода агента в шаблон отчёта — **10+ минут** на итерацию; JSON + слоты сокращают до **одного вызова скрипта** после review.
