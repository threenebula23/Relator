# Пример: отчёт по данным (JSON)

## Как это работает

1. Данные приходят из API, ETL или статического `data.json`.
2. Скрипт кладёт списки и скаляры в `Template.data()`.
3. Шаблон использует `[[TABLE...]]`, `[[LIST...]]`, циклы `%%len%%` по необходимости.

```bash
python build_report.py
```

Результат: `dist/business_report.md`.

## Пример вывода

```markdown
# Продажи Q1

## KPI по регионам

|region|revenue|
|---|---|
|EU|120k|
|US|95k|

## Заметки

- Запуск продукта B
- Рост EU +12%
```

## Почему это удобно

- Разделение: бизнес-логика в Python, оформление в Markdown-шаблоне.
- Повторное использование одного шаблона для еженедельных отчётов.

## Экономия времени

Копирование таблиц из Excel в документ **15–30 минут** в неделю; автоматизация сводит это к **запуску одной команды** после появления JSON.
